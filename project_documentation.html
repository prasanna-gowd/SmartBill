Chapter 1
Project Overview
SmartBill Pro is a desktop restaurant billing application that streamlines order management, menu configuration, and business analytics.
🎯 Problem Statement

Small to medium restaurants often rely on manual billing — handwritten orders, paper receipts, and no data insights. This leads to errors, slow service, lost revenue tracking, and no analytics. SmartBill Pro solves these problems with a digital billing system that handles the entire order lifecycle.

💡 Solution

A standalone desktop application that provides table-based ordering, a searchable menu, real-time cart with discount/tax calculations, receipt generation, comprehensive order history, full menu CRUD operations, visual analytics, and configurable settings — all backed by a local SQLite database requiring zero server setup.

Chapter 4
Features & Functionality
A detailed breakdown of every feature available in SmartBill Pro v2.0.
🔐

User Authentication

Login system with admin and staff roles. Each order is tracked to the logged-in user. Pre-configured default accounts.

📊

Live Dashboard

Real-time statistics cards showing today's revenue, order count, average order value, and total menu items. Recent orders list and quick action buttons.

🛒

POS / Billing System

Table-based ordering (1–20 tables). Browse 12 categories, search items live, add to cart with +/− quantity controls. Discount and tax calculations.

🔍

Menu Search

Live search across all menu categories. Results update as you type, making it fast to find items during peak hours.

💰

Discount & Tax

Configurable discount percentage per order. Tax rate set in settings (default 5%). Real-time total recalculation in the cart panel.

💳

Payment Methods

Choose between Cash, Card, or UPI for each order. Payment method is recorded and visible in order history and receipts.

🧾

Receipt Generation

Formatted receipt displayed after each order. Save receipts as text files in the reports/ folder for record keeping or printing.

📋

Order History

View all past orders with date range and table number filters. Double-click to see itemized details. Sort and browse efficiently.

📝

Menu Management

Full CRUD: Add new items, edit name/price/category, soft-delete items (preserved in DB). Filter by category and search by name.

📈

Analytics Dashboard

Four embedded matplotlib charts: Top Selling Items (bar), Monthly Revenue (bar), Category Breakdown (pie), Hourly Distribution (area).

📤

CSV Export

Export all order data to CSV with a file save dialog. Useful for external analysis in Excel or Google Sheets.

⚙️

Configurable Settings

Set restaurant name, tax rate, number of tables, and currency symbol. All settings persist in the database across sessions.

📐 Order Processing Formula
# Billing Calculation subtotal = sum(item.qty × item.price for item in cart) discount_amt = subtotal × (discount_pct / 100) after_discount = subtotal - discount_amt tax_amt = after_discount × (tax_rate / 100) total = after_discount + tax_amt # Example: Subtotal ₹1000, 10% discount, 5% tax # discount = ₹100, after_discount = ₹900, tax = ₹45 # TOTAL = ₹945

Chapter 9
Code Highlights
Key design patterns and techniques used in the codebase.
🏗️ Class-Based Architecture

The entire application is encapsulated in the SmartBillApp class, eliminating global variables and making state management clean.

main.py
class SmartBillApp: def __init__(self): self.root = tk.Tk() self.db = DatabaseManager() self.current_user = None self.current_table = None self.cart = {} # ... setup window, show login self.root.mainloop()
🔄 View Switching Pattern

Views are swapped by clearing the content frame and calling the appropriate builder method. Sidebar state updates automatically.

main.py
def switch_view(self, view_name): self.current_view = view_name # Update sidebar highlight for name, btn in self.sidebar_buttons.items(): color = Colors.SIDEBAR_ACTIVE if name == view_name else Colors.PRIMARY self._set_frame_bg(btn, color) # Clear content and load new view self.clear_content() views[view_name]()
🔀 Schema Migration

The database manager automatically upgrades older schemas by attempting to add missing columns — no manual migration needed.

db_manager.py
# Upgrade old databases automatically migration_columns = [ ("Orders", "subtotal", "REAL DEFAULT 0"), ("Orders", "discount_percent", "REAL DEFAULT 0"), ... more columns ... ] for table, col, col_type in migration_columns: try: cursor.execute(f"ALTER TABLE {table} ADD COLUMN {col} {col_type}") except sqlite3.OperationalError: pass # Column already exists
📊 Matplotlib Embedding

Charts are rendered directly inside Tkinter using FigureCanvasTkAgg, with graceful fallback if matplotib is not installed.

Password Hashing

Thermal Printer Support

Direct integration with POS receipt printers via ESC/POS protocol for physical receipt printing.

Convert to Flask/FastAPI + React web app for multi-device access across the restaurant.

Inventory Management

Track raw ingredients, set low-stock alerts, and auto-deduct inventory when orders are placed.
